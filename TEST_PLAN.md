# Markdown Assistant 测试计划

## 测试目标
验证 Markdown Assistant 应用的所有功能是否正常工作，包括：
- 基本编辑功能
- 三种编辑模式切换
- 文件操作功能
- 高级内容渲染（LaTeX、Mermaid等）
- 系统集成

## 测试环境
- 操作系统：Windows 10/11
- Node.js：18.0+
- Rust：1.70+
- Tauri：1.5+

## 测试步骤

### 1. 安装和启动
- [ ] 执行 `npm install` 安装依赖
- [ ] 执行 `npm run tauri:dev` 启动开发模式
- [ ] 验证应用是否成功启动，显示主界面

### 2. 基本编辑功能
- [ ] 在编辑器中输入普通文本
- [ ] 使用工具栏按钮格式化文本（粗体、斜体、列表等）
- [ ] 验证格式化效果是否正确
- [ ] 检查字符计数是否正常工作

### 3. 编辑模式切换
- [ ] 点击 WYSIWYG 按钮，验证切换到所见即所得模式
- [ ] 点击 IR 按钮，验证切换到即时渲染模式
- [ ] 点击 SV 按钮，验证切换到分屏预览模式
- [ ] 验证每种模式下的编辑体验是否正常

### 4. 文件操作
- [ ] 点击新建文件按钮，验证是否清空编辑器
- [ ] 输入内容后，验证修改指示器是否显示
- [ ] 点击保存按钮，验证文件保存对话框
- [ ] 选择位置保存文件，验证文件是否成功保存
- [ ] 点击打开文件按钮，选择已保存的文件，验证内容是否正确加载
- [ ] 测试快捷键：Ctrl+N、Ctrl+O、Ctrl+S、Ctrl+Shift+S

### 5. 高级内容渲染
- [ ] 测试数学公式：
  - 行内公式：$E = mc^2$
  - 块级公式：$$\frac{\partial f}{\partial x} = 2\sqrt{a}x$$
- [ ] 测试 Mermaid 图表：
  - 流程图
  - 甘特图
  - 时序图
- [ ] 测试代码块高亮：
  - JavaScript
  - Python
  - Rust
- [ ] 测试表格、引用、任务列表等其他 Markdown 功能

### 6. 系统集成
- [ ] 测试窗口大小调整
- [ ] 测试最小化/最大化窗口
- [ ] 测试应用关闭时的未保存提示

## 预期结果
- 所有功能均能正常工作
- 界面响应流畅
- 文件操作无错误
- 高级内容渲染正确
- 快捷键功能正常
